## 3. โครงสร้างข้อมูล (Firestore Schema)

```
clinics/{clinicId}
  ├─ name, lineChannelId, timezone
  ├─ services/{serviceId}      → name, durationMin, depositAmount
  ├─ quotas/{yyyy-mm-dd}       → { cash, sso, universal(บัตรทอง) } เพดานต่อวัน
  └─ slots/{yyyy-mm-dd}        → { time: capacity }

bookings/{bookingId}
  ├─ clinicId, patientLineId, patientName, phone
  ├─ serviceId, date, time
  ├─ coverage: "cash" | "sso" | "universal"
  ├─ status: "pending_slip" | "confirmed" | "reminded" | "done" | "no_show" | "cancelled"
  ├─ slip: { url, verified, amount, transRef, verifiedAt }
  └─ createdAt, updatedAt

patients/{lineId}   → name, phone, history[]
```

**Index ที่ต้องสร้าง:** `bookings` composite index `(clinicId, date, status)` สำหรับ dashboard และ reminder query.

## 4. ลำดับงานหลัก (Core Flows)

### 4.1 การจองคิว (Patient)
1. คนไข้เปิด LIFF จาก LINE OA → auth ด้วย LINE ID token
2. เลือกบริการ → ระบบ query `slots` + `quotas` เช็คว่าวัน/เวลานั้นยังรับสิทธิ์ที่เลือกได้
3. สร้าง booking สถานะ `pending_slip` (ล็อกที่นั่งชั่วคราว, TTL เช่น 15 นาที)
4. แสดงยอดมัดจำ + ช่องอัปโหลดสลิป

### 4.2 ตรวจสลิปอัตโนมัติ (SlipOK)
1. Upload รูป → Cloud Storage → ส่ง URL/ไฟล์ให้ SlipOK API
2. ตรวจ: ความถูกต้อง (กันสลิปปลอม), จำนวนเงิน ≥ ยอดมัดจำ, `transRef` ไม่ซ้ำ (กันใช้สลิปซ้ำ)
3. ผ่าน → `status: confirmed`, ตัดโควตาจริง, ส่ง Flex ยืนยัน + แจ้ง LINE Notify กลุ่มพยาบาล
4. ไม่ผ่าน → แจ้งคนไข้ให้ส่งใหม่ / ติดต่อแอดมิน

### 4.3 แจ้งเตือนกันลืมคิว (Reminder)
- Cloud Scheduler รันทุกวัน (เช่น 18:00) → query bookings `date == พรุ่งนี้ && status == confirmed`
- Push ข้อความเตือนผ่าน Messaging API → อัปเดต `status: reminded`

### 4.4 Dashboard คลินิก (Real-time)
- `onSnapshot` ฟัง `bookings` ของวันปัจจุบัน → แสดงตารางคิวแยกสีตาม `coverage` (ออกแบบสำหรับจอคอมพิวเตอร์)
- แอดมินตั้งเพดานโควตารายวันในหน้า Quota Settings → เขียนลง `quotas/{date}`

## 7. เกณฑ์ทดสอบก่อนส่งมอบ (Acceptance)

- จองซ้อนคิว/เกินโควตาไม่ได้ (concurrency-safe ด้วย Firestore transaction)
- สลิปปลอม / ยอดไม่ถึง / สลิปซ้ำ ถูกปฏิเสธ
- Dashboard อัปเดตภายใน ~1 วินาทีหลังคนไข้กดจอง
- แจ้งเตือนล่วงหน้า 1 วันส่งถึงคนไข้ทุกคิว confirmed
- Admin web app เปิดผ่านเบราว์เซอร์บนคอมพิวเตอร์ได้ปกติ (รองรับ Chrome/Edge/Safari)

## 8. หมายเหตุด้านความปลอดภัย

- ตรวจ LINE webhook signature ทุก request
- ยืนยัน LINE ID token ฝั่ง backend ก่อนไว้ใจ user
- Firestore rules: คนไข้อ่าน/เขียนได้เฉพาะ booking ของตัวเอง, แอดมินจำกัดตาม `clinicId`
- ไม่เก็บ transRef/ข้อมูลสลิปเกินจำเป็น, จำกัดสิทธิ์เข้าถึงรูปใน Storage
